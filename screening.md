## 1. Technical Fundamentals (Core Skills)

### Frontend – React
1. How would you structure a React application for a logistics dashboard (rates, orders, tracking)?
2. Explain how you manage state across multiple components.
3. What is the role of `useEffect`? Give a real-world example.
4. How do you handle API loading states, errors, and retries in React?
5. How would you design reusable form components (address, weight, dimensions)?

---

### Backend – Python / Flask
6. How would you structure a Flask project for a production-ready API?
7. Explain REST principles using a shipment or order example.
8. How do you validate request data and handle errors in Flask?
9. How would you design APIs for:
   - Creating shipments
   - Fetching rates
   - Tracking shipments
10. How do you handle long-running tasks (e.g., label generation, rate computation)?

## 2. Problem-Solving & System Thinking

11. Shipping rates change frequently. How would you design the system to handle this?
12. What core data would you store for each shipment?
13. How would you prevent duplicate shipment creation?
14. What happens if a carrier API is slow or unavailable?
15. How would you design a basic rate-comparison algorithm?

## 3. Logistics & Domain Understanding

16. What factors influence shipping cost and delivery time?
17. What are common reasons for shipment delays?
18. How would you design a shipment status timeline?
19. How would you explain a delay to a customer in simple terms?
20. What data would help improve customer experience in shipping?

## 4. AI / ML Awareness (Applied, Not Theoretical)

21. Where do you think AI can add value in logistics?
22. Have you used any LLMs (ChatGPT, OpenAI API, etc.) in a project? Explain.

24. What is prompt engineering? Why does it matter?
25. What are the risks of using AI in customer-facing systems?
